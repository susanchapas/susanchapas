# Gallery A11y Audit — 08

Scope: gallery/page.tsx, gallery/BentoTile.tsx, hooks/useMagneticTilt.ts, globals.css (reduced-motion).

## BLOCKERS

None. The disclosure pattern, keyboard operability, and reduced-motion gating are fundamentally sound.

## SHOULD-FIX

1. **Tablist is not a real tablist — missing tabpanel + keyboard model** — `page.tsx:49-86`
   `role="tablist"` with `role="tab"`/`aria-selected` is used, but: (a) there is no `role="tabpanel"` and no `aria-controls` linking the tabs to the grid, and (b) tabs lack the expected arrow-key roving-tabindex behavior — they remain plain Tab-stops. Screen-reader users are told "tab, 1 of N selected" but there's no panel relationship and no arrow navigation, which violates the ARIA tabs contract. Simplest fix: drop the tab roles and use a plain group of toggle buttons with `aria-pressed`, OR fully implement the panel + arrow keys. The half-implemented pattern is worse than buttons.

2. **Color contrast — `text-accent-lime` category eyebrow on `from-primary/90` gradient** — `BentoTile.tsx:128-131`
   Lime `#ccff00` on the dark navy gradient overlay passes AA easily for the lime itself, but the eyebrow is `text-[0.65rem]` (~10.4px) tracking-widest uppercase. It sits over a _gradient_ (`from-primary/90 to-transparent`); at the top of the title block the background is only ~90% opaque navy over the image, so contrast depends on the underlying photo. On light images the lime/title can drop below 4.5:1. Should-fix: the title `text-secondary` (white) at base/lg is the bigger risk on light photos — there's no solid scrim, only a gradient that fades to transparent. Consider a stronger/solid scrim behind the text band.

3. **Compact panel small text below comfortable AA** — `BentoTile.tsx:197-199`, `141`
   - Process step body `text-secondary/70 text-xs` (~12px). `text-secondary` at 70% on `bg-primary` navy: white@0.70 over navy yields roughly ~6–7:1 — passes AA 4.5:1, but it's the weakest text and is small. Acceptable, monitor.
   - `text-secondary/60` "Tap to view the process" (`:141`, ~12px) is the lowest — white@0.60 over navy is near ~5:1, still ≥4.5 but with little margin. Nice-to-have to bump to /70.
     The description `text-secondary/75 text-sm` (`:178`) is fine.

4. **Sibling-dimming is mouse-hover only — keyboard/focus users never trigger or escape it correctly** — `BentoTile.tsx:112-118`, `page.tsx:100`
   The dim overlay keys off `group-hover/grid` (lights up all siblings to `opacity-55`) and `group-hover/tile:!opacity-0` (un-dims the hovered one). This is `:hover`-only — keyboard focus does **not** activate it. Net effect:
   - Keyboard users get NO dimming at all (acceptable — no harm, the effect is decorative).
   - But it means the focused tile gets no visual emphasis from this system; focus relies solely on the browser default outline on the inner `<button>`. Confirm a visible focus ring exists (not overridden). Should-fix: add `group-focus-within/tile:!opacity-0` so keyboard focus also clears the dim if the grid is ever dimmed via focus, and/or ensure a strong `:focus-visible` ring on the button.
   - The overlay is `aria-hidden` and `pointer-events-none` (good — no AT/interaction leakage). The dim sits _behind_ the title gradient band (`z-10` on the text block vs no z on dim), so title text is not additionally darkened by it — contrast of title is unaffected by dimming. Good.

## NICE-TO-HAVE / CONFIRMED OK

5. **Inline disclosure pattern — correct.** `BentoTile.tsx:82-88` button has `aria-expanded={expanded}` + `aria-controls={detailId}`; the panel `motion.div` at `:151-152` carries the matching `id={detailId}`. When collapsed, `AnimatePresence` + `{expanded && ...}` (`:149-150`) removes the panel from the DOM entirely — correct, no hidden focusable content. `aria-label` swaps View/Collapse (`:87`). Solid disclosure.

6. **Focus management on collapse — minor gap.** Close button (`:209-216`) is inside the panel. When clicked, `handleToggle` collapses and the panel (containing the focused close button) unmounts. Focus is then lost to `<body>`. Should-ideally return focus to the tile's trigger button. Low severity since the trigger is the immediately-preceding sibling, but flagging: keyboard users lose their place. Nice-to-have: ref the trigger and `.focus()` it on collapse.

7. **Keyboard operability — good, no traps.** Both the tile trigger (`:82`) and close button (`:209`) are native `<button>` — Enter/Space work natively. No mouse-only interaction for opening/closing (the tilt/glare are pure decoration gated to fine-pointer mouse, `useMagneticTilt.ts:70-77` early-return on non-mouse pointerType). Close button is reachable via Tab. No keyboard trap.

8. **Reduced-motion — well gated.**
   - Hook: `useMagneticTilt.ts:33-35` `enabled = finePointer && !reduce && !disabled`; when reduced, `tiltStyle`/`imgStyle` are `undefined` and `handlers` is `{}` — tilt + glare motion-values + boxShadow all disabled. Good.
   - Tile entrance: `variants={reduce ? undefined : tileVariant}` (`:68`) — no blur/translate/scale entrance under reduce. Good.
   - Expand height-animation: `initial={reduce ? false : {height:0,...}}` and `exit={reduce ? {opacity:0} : {...}}` (`:153-155`) — collapses to opacity-only, no height tween. Good.
   - Page-level hero + filter pill: `m(reduce, ...)` and `initial={reduce ? false : ...}` (`page.tsx:24-26, 75, 94`). Good.
   - The glare overlay (`BentoTile.tsx:120-127`) is CSS opacity `transition` on `group-hover/tile` — globals.css `:44-50` zeroes `transition-duration` under reduce, AND the hover handlers are removed under reduce so `--mx/--my` never update. Effectively gated. OK.
   - No ungated JS motion found.

9. **Media — no captions.** `ArtMedia` videos: tiles pass `alt`/`aria-label` (`BentoTile.tsx:99-108, 161-167`) but there is no `<track kind="captions">`. For the EOP explainer video this is a WCAG 1.2.2 (Captions, prerecorded) gap if the video has speech/narration. Note for that specific asset — add captions track. (Decorative/silent loops are exempt.)

## Summary priority

1. Fix or remove the partial tablist pattern (#1).
2. Verify title/eyebrow contrast over light photos — add solid scrim (#2).
3. Ensure a strong `:focus-visible` ring on tile buttons; consider focus-within un-dim (#4).
4. Return focus to trigger on panel collapse (#6).
5. Captions track for the EOP explainer video (#9).
